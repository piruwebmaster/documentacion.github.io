    Introducción

    La presente documentación hace referencia al conjunto de Endpoint disponibles en los procesos de operaciones sobre Pólizas que van desde la Cotización hasta la Cancelación para ser consumidos por aplicaciones Externas. Los endpoint relacionados con las cotizaciones y pólizas basan su funcionalidad y operatividad en la configuración de los productos de Seguros Universal bajo las especificaciones de un motor de tarifa dinámico para la resolución de los valores y cálculos de las coberturas por producto y planes. Dicho motor de tarifa centra su estructura en una definición de datos particulares por productos y planes llamados (Métodos y Componentes). Los “Métodos” viene a ser los Datos (Campos o Informaciones de Pólizas, Asegurados y Bienes) que se defines según la naturaleza de un Producto. Los posibles valores de un Método se conocen como “Componentes” (Valores de Métodos).

    Crear una propuesta

    Descripción

    El endpoint {{baseUrl}}/api/Cotizador/Propuesta de tipo POST permite crear propuesta dado un diccionario de clave valor de tipo string.

    Existen ciertas convenciones para fechas y beneficiarios, las fechas deben ser proporcionadas en el formato dd/MM/yyyy. Las cantidades deben ser delimitadas entre comillas dobles por ejemplo una propiedad SumaAsegurada con valor de 1000 debe ser enviada {"SumaAsegurada":"1000"}.

    A su vez existen propiedades que deben de estar presentes en la petición las que se ejemplifican a continuación.

    Ejemplos de entrada

    {
  "codProd": "F-BD",
  "codPlan": "001",
  "revPlan": "001",
  "codRamo": "VITR",
  "codMoneda": "RD",
  "codUsr": "externo112",
  "tipoPrima": "LLAMATIVA",
  "tipoPropuesta": "abc",
  "fecIniVigPropuesta": "17/03/2023",
  "fecFinVigPropuesta": "17/03/2024",
  "MostrarComo": null,
  "descripcionBienAsegurado": " nombre descriptivo "
}

    La propiedad "MostrarComo" tiene un valor por defecto null, al enviarse el string "Array" configura la propuesta como multibien debiendo enviarse la propiedad "bienesAsegurados" de tipo array tal como sigue

    {
//...
    "MostrarComo": "Array",
    "bienesAsegurados": [
    {
     //...
        "AnioFabricacion": "2023",
        "descripcionBienAsegurado": "Cámara",
        "DescripcionObjetoAsegurado": "Cámara",
     //...
    }
}

    Cada entrada de la propiedad bienes asegurados debe ser un objeto que contenga el nombre de la propiedad (Método) del objeto asegurado en cuestión.

    Si el producto contempla beneficiarios es necesario enviar la propiedad beneficiarios de tipo array en el cuerpo principal de la petición.

    {
  //...
  "beneficiarios": [
    {
      "numOa": 1,
      "beneficiarioNombreCompleto": "DARITZA CAROLINA MATOS DE LOS SANTOS",
      "beneficiarioTipoIdentificacion": "Cédula",
      "beneficiarioNumeroIdentificacion": "___-_______-_",
      "beneficiarioPorcentaje": 100,
      "beneficiarioRelacion": "Otra relación",
      "beneficiarioFechaDeNacimiento": "29/05/1976",
      "numBen": 1
    }
  ]
  //...
}

    El endpoint {{baseUrl}}/api/Cotizador/Propuesta de tipo POST devuelve las mismas informaciones enviadas en la petición adicionando el ideprop Generado y las informaciones de primaAnual primaAnualSinImpuestos iscAnual impuestoAnual

    Descripción

    El enpoint de tipo {{baseUrl}}/api/ropuesta/{ideprop}/DatosEmision de tipo PUT recibe un diccionario de tipo string -> string que contiene las propiedades y sus respectivos valores a editar, su utilidad radica en que la alteración de estas propiedades tiene una tiempo de ejecución más corto al no ser necesario realizar computos de prima, fraccionamiento ni impuestos.

    Cancelar Póliza

    Descripción

    El endpoint {{baseUrl}}/api/poliza/{codprod}-{numpol}/cancelar de tipo POST recibe un objeto del tipo.

    Ejemplo de entrada

    {
  "fechaCancelacion": "2023-03-07",
  "motivoCancelacion": "01",
  "tipoAnulacion": "P",
  "indDevPrima": "S",
  "DescripcionesMotvAnul": "El producto no cumple con mis expectativas"
}

    Con las información relativas a la cancelación, el parámetro motivoCancelacion, tipoAnulacion y indDevPrima recibe los siguientes valores.

    Motivos de Cancelación

    Valor Descripción
    01 POR FALTA DE PAGO
    02 CANCELACIÓN POR DECISIÓN DE UNIT
    03 CANCELACIÓN POR SINIESTRO
    04 SUSTITUCIÓN COBERTURA ACTUAL
    05 EL PRODUCTO NO CUMPLE CON MIS EXPECTATIVAS
    06 TUVE UNA MALA EXPERIENCIA / MAL SERVICIO AL CLIENTE
    07 ENCONTRÉ UNA MEJOR ALTERNATIVA
    08 YA NO PUEDO SEGUIR PAGANDO LA PRIMA
    09 SUSTITUÍ MI COBERTURA POR OTRA COBERTURA DE UNIT
    10 VENDÍ MI PRODUCTO
    11 PRUEBA REALIZADA

    Tipos de Anulación

    Valor Descripción
    P Prorrata
    C Corto Plazo

    Indicador Devuelve Prima

    Valor Descripción
    S Devuelve Prima
    N No Devuelve Prima

    Ejemplo de Respuesta

    Devuelve un objeto del tipo.

    {
  "data": "resultado"
}
